Pol Espinasa 24/04/2026 16:13

Se mira si se hara un relay inmediato de la transaccion. Se hara relay inmediato en caso que el backlog global este vacio, haya tokens suficientes y no se haya consumido la memoria reservada.
Relay inmediato significa decremantar el numero de tokens y reducir el tamaño de memoria disponible y hacer push de la transaccion al m_tx_inventory_to_send de cada peer. En caso que no se haga relay immediato se hace push de la transaccion al backlog global.

Cada ciclo se llama a CatchupRelayTransactions(), ahi se hace refill de tokens según el tiempo transcurrido. Si hay suficientes tokens, se hace push de las mejores transacciones ordenadas por feerate, teniendo en cuenta el AncestorSet, al m_tx_inventory_to_send de cada peer.

Cada N segundos (dependiendo de si es inbound o outbound) se ordena la m_tx_inventory_to_send de cada peer y se manda al peer.
