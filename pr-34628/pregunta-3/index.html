nebula 24/04/2026 16:17
Sirve para controlar la velocidad al a que ocurren acciones, el cubo se llena con tokens con un maximo fijado y cada acción consume un token. Si hay tokens todo normal, si no hay, se retrasa. El dual bucket de este PR consiste en un cubo que limita el num de tx y el otro el tamaño total. Creo que el primer cubo es para frenar la saturación de la CPU y el segundo la memoria.
